Missing {doc_key} documentation may represent a **clinical safety requirement** (e.g., latent TB screening before immunosuppression, hepatitis screening before biologics). Your analysis identifies **payer decision patterns** — correlations in historical approval/denial data. These patterns are useful for **PA submission strategy** (what to emphasize in the submission), but:

- **NEVER state or imply that clinical factors "compensate for" or "replace" missing safety screenings.** Missing screening documentation is a clinical gap regardless of payer decision patterns.
- **Frame findings as submission strategy insights**: "Payers have historically approved cases with this gap when severity markers were high — include severity documentation prominently in submission."
- **Always recommend pursuing the missing documentation**: Every overall_summary MUST end with a statement that the missing {doc_key} should still be obtained.

## Task

Among patients who were ALL missing {doc_key} documentation, identify what clinical factors **correlate with** approval vs denial. Focus on:

- **Clinical markers associated with approval** (e.g., "Cases with CRP >= 40 were approved at 85% vs 30% among those with CRP < 40 — higher disease severity correlates with approval despite this gap")
- **Specific numeric inflection points** (lab values, treatment counts, timing windows)
- **Multi-variable interactions** (e.g., high CDAI + elevated CRP together correlate with approval) — ONLY report these if BOTH sub-cohorts have >= 5 cases; with smaller cohorts, multi-variable interactions are unreliable noise
- **Other documentation that correlates with approval** (e.g., having vaccination records present correlates with approval despite missing Hep B/C screening)

For each finding, assess where the CURRENT patient stands relative to this differentiator.

## Statistical Validity Rules

- Sub-cohort sizes: approved={approved_count}, denied={denied_count}. Total resolved={total_missing_cases}.
- **If either sub-cohort has < 5 cases**: Findings are preliminary observations, NOT reliable thresholds. Evidence strength MUST be "weak". Do NOT claim specific thresholds from <5 cases.
- **If both sub-cohorts have >= 8 cases**: Findings can be presented as moderate-to-strong patterns.
- **For hidden_patterns (multi-variable interactions)**: ONLY report if BOTH sub-cohorts have >= 5 cases. With fewer cases, multi-variable slicing produces noise. Return an empty array if sub-cohorts are too small.

## Output Rules

1. **Insights**: Return AT MOST 4. Each `headline` must be ≤60 characters and follow the format "[Differentiator]: [key number]" (e.g., "CRP above 40: 85% approved vs 30%"). Each `finding` is 1-2 short sentences framed as **payer decision pattern** (correlation), NOT clinical compensation. Each finding must be understandable on its own — state which sub-cohort you mean, name the metric plainly, and give the exact number. Each `current_patient_detail` is one sentence, ≤80 characters, stating the patient's position directly (e.g., "Patient CRP is 42 — above the 40 threshold" or "Patient does not have fecal calprotectin on file").
2. **Recommendations**: Return AT MOST 3. Every insight marked `at_risk` MUST have a corresponding recommendation. Each `action` is ≤12 words. Each `expected_impact` must reference cohort data.
3. **Patient Position**: `overall_summary` is max 2 sentences assessing submission positioning, PLUS a final sentence recommending the missing documentation still be obtained. Cap `favorable_factors` and `at_risk_factors` at 3 each.
4. **Hidden Patterns**: Return AT MOST 2. ONLY if both sub-cohorts have >= 5 cases. Otherwise return empty array.
5. **Defensibility**: Every percentage or number you cite MUST come directly from the cohort statistics provided. No extrapolation, no invented numbers.
6. **Non-obvious only**: Each insight must have `non_obvious_factor: true`. Do NOT surface obvious findings like "having more documentation correlates with approval."
